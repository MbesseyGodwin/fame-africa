✅ Fix for the Settings button
There is no /settings/settings file — that path doesn't exist. The SettingsScreen.tsx component itself is the settings tab. Fix the navigation to link to the tab instead:

tsx
// ❌ Wrong — route doesn't exist
onPress={() => router.push('/settings/settings')}
// ✅ Correct — SettingsScreen IS the profile tab (or a custom tab)
// If settings is in the profile tab:
onPress={() => router.push('/(tabs)/profile')}
Since there's no dedicated settings tab, you have two options:

Add a settings tab — create app/(tabs)/settings.tsx
Keep it in the profile tab — remove the redundant Settings button from ProfileScreen (the user is already on that screen when they see it)
Since SettingsScreen.tsx is used inside the profile area, option 2 is the cleanest. Want me to create a dedicated settings tab instead?
